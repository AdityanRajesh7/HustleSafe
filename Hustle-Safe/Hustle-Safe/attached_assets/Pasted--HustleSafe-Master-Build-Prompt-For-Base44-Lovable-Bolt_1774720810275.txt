## PROJECT OVERVIEW

Build **HustleSafe** — a full-stack, production-ready, MVP web application. HustleSafe is a real-time AI-driven parametric income protection platform for India's food delivery gig workers (Zomato/Swiggy partners). It is NOT a traditional insurance claims tool — it is an intelligent, automatic income protection system that detects disruptions, activates coverage pre-emptively, and releases payouts without the worker filing any claim.

**Hackathon Context:** Guidewire DEVTrails 2026 — University Hackathon. This must be polished, impressive, and win-ready.

  id TEXT PRIMARY KEY,

  govt_alert BOOLEAN DEFAULT false,
  last_updated TIMESTAMPTZ DEFAULT NOW()

-- Claims / Payouts
CREATE TABLE claims (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  worker_id UUID REFERENCES workers(id),

  disruption_start TIMESTAMPTZ,

  hourly_rate DECIMAL(8,2) DEFAULT 90,
  payout_amount DECIMAL(10,2),
  fraud_score DECIMAL(3,2) DEFAULT 0.0,

-- Disruption Events (the simulator writes here)
CREATE TABLE disruption_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  duration_minutes INTEGER,
  triggered_by TEXT DEFAULT 'simulator',
  is_active BOOLEAN DEFAULT false,
  started_at TIMESTAMPTZ,
  ended_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()

-- Premium History
CREATE TABLE premium_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  worker_id UUID REFERENCES workers(id),
  week_start DATE,
  premium_amount DECIMAL(8,2),
  zone_risk_adjustment DECIMAL(8,2),
  worker_risk_adjustment DECIMAL(8,2),

**Simulator Logic (CRITICAL — must work end-to-end):**
When "TRIGGER DISRUPTION" is clicked:
1. Insert a record into `disruption_events` table with `is_active = true`
2. Update the target `zones` row: set `gds_score` to GDS target, `status` to appropriate level, update `rainfall_mm`/`traffic_score`/etc. based on event type, set `last_updated = NOW()`
3. Supabase Realtime broadcasts the zone change to ALL connected clients
4. A background timer (setInterval in Zustand store, `duration_minutes × 60` seconds) counts down
5. Auto-generate claims for workers in that zone: query workers with active policies in the zone → for each worker, compute payout (hours × ₹90) → insert into `claims` table with `status = 'auto_approved'` (or 'soft_hold' for ~15% randomly)
6. On the worker dashboard (if logged in as a worker in that zone), the alert banner appears immediately via Realtime
7. When duration ends (or "END ALL EVENTS" clicked): update zone back to normal GDS (15-25), set `is_active = false` on the event, update worker statuses

## REALTIME ARCHITECTURE (CRITICAL)

```javascript
// Zustand store handles all realtime state
// On app load, subscribe to:

supabase
  .channel('zone-changes')
  .on('postgres_changes', 
    { event: 'UPDATE', schema: 'public', table: 'zones' },
    (payload) => {
      useZoneStore.getState().updateZone(payload.new)
    }
  ).subscribe()

supabase
  .channel('claim-changes')
  .on('postgres_changes',
    { event: 'INSERT', schema: 'public', table: 'claims' },
    (payload) => {
      useClaimStore.getState().addClaim(payload.new)
      // If claim zone matches logged-in worker's zone → show alert
    }
  ).subscribe()
```

The Zustand `zoneStore` holds all 8 zone objects. When any zone updates via Realtime, all components reading from the store re-render immediately — no polling needed.

## DISRUPTION SIMULATOR — DETAILED LOGIC

This is the CENTERPIECE of the demo. It must work flawlessly.

```typescript
// simulatorStore.ts (Zustand)

interface SimulatorState {
  isRunning: boolean;
  activeEvent: DisruptionEvent | null;
  countdown: number; // seconds remaining
  affectedWorkers: number;
  generatedClaims: Claim[];
}

async function triggerDisruption(params: {
  zoneId: string;
  eventType: EventType;
  gdsTarget: number;
  durationMinutes: number;
}) {
  // 1. Insert into disruption_events
  const event = await supabase.from('disruption_events').insert({...params, is_active: true, started_at: new Date()})

  // 2. Update zone GDS and signals
  const signals = computeSignalsFromEventType(params.eventType, params.gdsTarget)
  await supabase.from('zones').update({
    gds_score: params.gdsTarget,
    status: gdsToStatus(params.gdsTarget),
    ...signals,
    last_updated: new Date()
  }).eq('id', params.zoneId)
  // ↑ This triggers Supabase Realtime → ALL clients update instantly

  // 3. Find affected workers and generate claims
  const workers = await supabase
    .from('workers')
    .select('*, policies(*)')
    .eq('zone_id', params.zoneId)
    .eq('is_active', true)

  // Generate claims for each worker with active policy
  for (const worker of workers.data) {
    if (!worker.policies?.length) continue;
    const fraudScore = generateFraudScore(worker) // 0.05–0.35 for legit, spike for demo fraud case
    const hoursAffected = params.durationMinutes / 60
    const payout = Math.min(hoursAffected * 90, worker.policies[0].coverage_cap)
    const status = fraudScore > 0.72 ? 'insurer_review' : fraudScore > 0.40 ? 'soft_hold' : 'auto_approved'
    
    await supabase.from('claims').insert({
      worker_id: worker.id,
      policy_id: worker.policies[0].id,
      zone_id: params.zoneId,
      disruption_type: params.eventType,
      disruption_start: new Date(),
      hours_affected: hoursAffected,
      payout_amount: payout,
      fraud_score: fraudScore,
      status,
      fraud_signals: generateFraudSignals(fraudScore)
    })
    // ↑ Each insert triggers Realtime → claims feed updates on insurer dashboard
  }

  // 4. Start countdown timer
  const timer = setInterval(async () => {
    // decrement countdown, update store
    if (countdown <= 0) {
      clearInterval(timer)
      await endDisruption(params.zoneId, event.data.id)
    }
  }, 1000)
}

function computeSignalsFromEventType(eventType: string, gds: number) {
  const maps = {
    heavy_rain: { rainfall_mm: 35 + gds * 0.3, traffic_score: 7.5, demand_drop_pct: 55 },
    flood: { rainfall_mm: 65 + gds * 0.2, traffic_score: 9.2, demand_drop_pct: 80, govt_alert: true },
    curfew: { govt_alert: true, demand_drop_pct: 90, traffic_score: 8.0, rainfall_mm: 0 },
    platform_outage: { demand_drop_pct: 95, traffic_score: 2.0, rainfall_mm: 0 },
    strike: { demand_drop_pct: 70, govt_alert: true, traffic_score: 6.5, rainfall_mm: 0 },
    aqi_hazard: { aqi: 280, demand_drop_pct: 40, traffic_score: 5.0, rainfall_mm: 0 }
  }
  return maps[eventType] || {}
}
```
